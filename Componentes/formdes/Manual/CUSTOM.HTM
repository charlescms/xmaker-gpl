Custom transitions

To create a custom transition you have to derive a new class from one of the available classes. The root class is TTransitionEffect. It handles all the basics to perform transitions, but it is completely inoperative by itself. All the transitions derive directly or indirectly from it. In order to configure your custom transitions you may have to override some virtual methods, depending on your needs. This methods are:

  • DoExecute
  • GetPixelFormat
  • NeedDstImage
  • NeedSrcImage
  • RenderWhenClipped
  • TwoPassesCapable
  • UseOffScreenBmp

As an example, we have taken the TShakeTransition of the Demo project.

If you want to CPU-independently time your transition you can derive it from TTimedTransitionEffect instead. This way you can take advantage of its built in time handling algorithm. In this case you don't have to override DoExecute because you will use TTimedTransitionEffect's one. Instead, you have to override some of the following methods:

  • ExecuteFrame
  • Finalize
  • Initialize

An example of this is the TSimpleWipeTransition of the Demo project.

If you have created nice transitions we could put them into our website.

We are planning to develope many more transitions. If you don't like to create your own you can always request it to us. We will give preference to the most requested and interesting ones.
