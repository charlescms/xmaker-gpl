teNoRender, teAuto, tePaint, tePrint, teEmulate

Unit

teRender

Description

These consts indicate how to render a control:
  • teNoRender: The control will not be rendered at all. It should be refreshed once the transition has finished.
  • teAuto: The proper render mode will be automatically detected (it may not work with some controls).
  • tePaint: The control will be rendered using a paint approach.
  • tePrint: The control will be rendered using a print approach.
  • teEmulate: The control will be rendered using an emulation approach.
